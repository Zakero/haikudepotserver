Tento proces přijme archivní soubor tar, který byl komprimován gzip. Typické přípona souboru pro tento formát archivu je ".tgz", ale přípona souboru Někdy se také používá ".tar.gz". Soubory v archivu by měly mít příponu následující nomenklatura; "hicn/beshare/icon.hvif". V tomto příkladu HVIF vektorová ikona je dodávána pro balíček "beshare".

Někdy nejsou k dispozici data vektorových ikon. V tomto případě kombinace 16, 32 a 64 mohou být dodány obrázky PNG se čtvercovými pixely. V tomto případě následující archivní záznam byla by použita nomenklatura; "hicon/beshare/16.px".

Pro každý balíček, který má ikony upravené tímto procesem, budou data předchozí ikony odstraněno.

Import tar souboru ikon je "atomový"; což znamená, že bude buď zpracuje archiv úplně, nebo nezmění data na straně serveru. Toto je in ve vztahu k runtime chybám, jako jsou neplatná vstupní data, která brání procesu běží vůbec.

Jsou hlášeny problémy, jako jsou chybějící balíčky nebo úspěšné načtení ikony zpět operátorovi prostřednictvím vygenerovaného souboru CSV.
