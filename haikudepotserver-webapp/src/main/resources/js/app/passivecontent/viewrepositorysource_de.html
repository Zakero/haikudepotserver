Das Repository gibt es für verschiedene Architekturen. Für jede gibt es eine eigene "Repository-Quelle". Die Repository-Quelle hat mehrere Spiegelserver, einer davon wurde zum primären Spiegelserver gemacht, vom dem die Daten kommen um den HaikuDepotServer zu aktualisieren.

Neben den eigentlichen Repository Daten, existiert noch eine repo.info Datei mit einem identifier Feld. Dies ist ein eindeutiger Identifikator für die eigentliche Repository-Quelle. Dieser identifier verknüpft die HDS Daten mit dem in der HaikuDepot Anwendung konfigurierten Repository.
